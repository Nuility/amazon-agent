# CopilotKit + Wimoor 广告投放模块整合包总结

## 一句话说明

这个整合包的作用，是把 Wimoor ERP 中的亚马逊广告投放业务能力，与 CopilotKit 的 agent UI / 人机协同能力提前整理成一个可接入的中间包，方便后续融合到你的广告 agent 项目中。

## 为什么这样整合

Wimoor 是完整 ERP，业务范围很大。如果直接整体合并，会把订单、库存、采购、财务、后台权限、菜单体系等大量非广告内容带进来，增加后续 agent 项目的复杂度。

因此本包只围绕广告投放闭包整理：

- 广告活动、广告组、关键词、投放对象、预算、竞价等广告管理能力。
- 广告报表、广告概览、广告表现指标和趋势分析能力。
- 店铺、站点、profile、Amazon 广告授权等运行广告模块所需上下文。
- 面向 agent 的查询、分析、建议和受控执行接口。

CopilotKit 则不作为业务系统源码合并，而是作为后续 agent 前端体验的参考和依赖来源。它负责让用户通过聊天、页面上下文、生成式 UI 和确认流程与广告 agent 协作。

## 包里包含什么

- `wimoor-advertising/`：Wimoor 广告模块的边界、源码清单和广告接口契约笔记。
- `copilotkit-agent-ui/`：CopilotKit 在广告 agent 场景中的接入方式和保留/排除清单。
- `contracts/`：后续 agent 项目可直接参考的工具契约和 UI 事件契约。
- `handoff/agent-fusion-blueprint.md`：后续真正融合进 agent 项目时的实施路线。
- `SOURCES.md`：上游项目来源、当前本地获取状态和网络恢复后的源码补齐命令。

## 后续融合时的推荐路线

第一阶段先做只读分析：广告活动列表、广告报表、账户概览、异常指标解释。

第二阶段做优化建议：根据 ACOS、ROAS、CTR、CPC、CVR、花费、订单等指标生成预算、竞价、关键词和投放对象建议。

第三阶段做受控执行：暂停广告、调整预算、调整竞价、新增/否定关键词等动作必须先让用户确认，并记录审计日志和回滚提示。
