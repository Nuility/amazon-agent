### 五、遇到的问题与解决

#### 问题1: pip命令未找到

**原因**: Python安装但未添加到PATH环境变量

**解决**: 
- 使用完整Python路径: `%LOCALAPPDATA%\Programs\Python\Python314\python.exe`
- 创建启动脚本自动检测路径
- 文档中说明环境变量配置方法

#### 问题2: 端口8000被占用

**原因**: 系统已有服务占用8000端口

**解决**:
- 修改启动脚本使用8001端口
- 文档中说明端口配置方法
- 添加端口检查和切换说明

#### 问题3: 模块导入失败

**原因**: API文件在子目录，路径引用错误

**解决**:
```python
# 修复前
src_dir = os.path.dirname(os.path.abspath(__file__))

# 修复后
api_dir = os.path.dirname(os.path.abspath(__file__))
src_dir = os.path.dirname(api_dir)
```

## 备注

### 关键技术点

1. **FastAPI生命周期**: 使用`@asynccontextmanager`管理应用启动和关闭
2. **CORS配置**: 允许跨域访问，支持前后端分离
3. **Pydantic模型**: 请求参数验证和类型安全
4. **CSS变量**: 便于主题定制和维护
5. **响应式设计**: 适配不同屏幕尺寸

### 性能考虑

1. 使用异步处理提高并发性能
2. 前端事件防抖减少请求频率
3. 分页加载减少数据传输
4. CSS变量减少样式计算

### 安全考虑

1. CORS已配置但建议生产环境限制来源
2. 未实现认证，建议接入JWT
3. 敏感操作需要二次确认
4. 日志记录所有操作
