## 一、架构概览

### 1.1 系统定位

Amazon Ads Agent 是一个智能化广告优化系统，集成了大模型AI能力，提供Amazon广告活动的智能分析、优化建议生成和自动化执行能力。

### 1.2 架构模式

采用**严格的分层架构**，遵循依赖倒置原则和SOLID设计原则：

```
┌─────────────────────────────────────────────────────────┐
│                    接口层 (Interface)                     │
│                CLI / FastAPI REST API                    │
├─────────────────────────────────────────────────────────┤
│                    服务层 (Services)                      │
│         业务逻辑、规则引擎、智能分析、AI决策               │
├─────────────────────────────────────────────────────────┤
│                  仓储层 (Repositories)                    │
│              数据访问抽象、查询封装                        │
├─────────────────────────────────────────────────────────┤
│               基础设施层 (Infrastructure)                 │
│      存储、日志、LLM客户端、Amazon Ads客户端              │
├─────────────────────────────────────────────────────────┤
│                  公共层 (Common)                         │
│          数据模型、异常、校验器、工具函数                  │
└─────────────────────────────────────────────────────────┘
```

#### 3.5.2 广告域模型（ad_types.py）

**广告报告模型**:

```python
@dataclass
class AdReportRecord:
    """广告报告记录"""
    campaign_id: str
    campaign_name: str
    date: str
    impressions: int          # 展示次数
    clicks: int               # 点击次数
    cost: float               # 花费
    orders: int               # 订单数
    sales: float              # 销售额
    status: str
    tags: List[str]
    
    @property
    def ctr(self) -> float:
        """点击率 CTR = clicks / impressions"""
        return self.clicks / self.impressions if self.impressions > 0 else 0
    
    @property
    def cvr(self) -> float:
        """转化率 CVR = orders / clicks"""
        return self.orders / self.clicks if self.clicks > 0 else 0
    
    @property
    def cpc(self) -> float:
        """单次点击成本 CPC = cost / clicks"""
        return self.cost / self.clicks if self.clicks > 0 else 0
    
    @property
    def acos(self) -> float:
        """广告成本销售比 ACOS = cost / sales"""
        return self.cost / self.sales if self.sales > 0 else 0
    
    @property
    def roas(self) -> float:
        """广告投资回报率 ROAS = sales / cost"""
        return self.sales / self.cost if self.cost > 0 else 0
```

### 9.1 存储适配器扩展

**接口**: `StorageAdapter`

**扩展示例**:

```python
class PostgreSQLStorageAdapter(StorageAdapter):
    """PostgreSQL存储适配器"""
    
    def __init__(self, connection_string: str):
        self.engine = create_engine(connection_string)
        self.Session = sessionmaker(bind=self.engine)
        
    def save(self, entity_type: str, entity_id: str, data: Dict) -> bool:
        session = self.Session()
        try:
            # 保存到数据库
            session.commit()
            return True
        except:
            session.rollback()
            return False
        finally:
            session.close()
```

### 9.2 LLM客户端扩展

**接口**: `LLMClient`

**扩展示例**:

```python
class CustomLLMClient(LLMClient):
    """自定义大模型客户端"""
    
    def __init__(self, config: Dict[str, Any]):
        self.api_key = config['api_key']
        self.endpoint = config['api_endpoint']
        
    def call(self, prompt: str, **kwargs) -> str:
        # 实现API调用逻辑
        response = requests.post(
            self.endpoint,
            headers={'Authorization': f'Bearer {self.api_key}'},
            json={'prompt': prompt}
        )
        return response.json()['result']
        
    def is_available(self) -> bool:
        # 检查API可用性
        try:
            self.call("test")
            return True
        except:
            return False
```

## 十二、总结

### 12.1 架构优势

1. **清晰的分层架构** - 职责明确，易于维护和测试
2. **依赖注入** - 组件解耦，支持灵活替换
3. **多LLM支持** - 可切换不同大模型提供商
4. **完整的API设计** - RESTful风格，自动文档生成
5. **类型安全** - 全面使用Python类型注解
6. **双重交互方式** - CLI命令行 + Web图形界面
7. **配置灵活** - 支持YAML/JSON/环境变量多源配置

### 12.2 核心创新点

1. **AI驱动的广告优化** - 集成大模型生成优化建议
2. **闭环执行系统** - 建议生成 → 审核 → 执行 → 反馈
3. **双模式运行** - 生产模式和试运行模式（dry_run）
4. **智能降级策略** - LLM不可用时自动降级为规则引擎
5. **完整审计追踪** - 所有操作记录可追溯
