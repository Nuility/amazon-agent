### 2.2 架构概览

#### 分层架构

```
┌─────────────────────────────────────┐
│     Interface Layer (CLI/API)       │  ← 用户交互入口
├─────────────────────────────────────┤
│      Service Layer (业务逻辑)        │  ← 核心业务处理
├─────────────────────────────────────┤
│   Repository Layer (数据访问)        │  ← 数据操作抽象
├─────────────────────────────────────┤
│ Infrastructure (基础设施)           │  ← 存储、日志、外部API
└─────────────────────────────────────┘
```

#### 模块划分

| 层级 | 模块 | 文件数 | 说明 |
|------|------|--------|------|
| **API层** | api/ | 4 | FastAPI服务、启动脚本 |

| **仓储层** | repositories/ | 6 | 数据访问抽象 |
| **基础设施层** | infrastructure/ | 6 | 存储、日志、客户端 |
| **公共层** | common/ | 5 | 数据模型、异常、工具 |

### 2.4 架构特点

#### 设计模式应用

| 模式 | 应用场景 | 说明 |
|------|----------|------|
| 仓储模式 | 数据访问层 | 抽象数据访问 |
| 适配器模式 | 存储适配器、LLM客户端 | 统一接口 |
| 工厂模式 | LLM客户端创建 | 根据配置创建实例 |
| 策略模式 | LLM降级策略 | 自动降级 |
| 依赖注入 | 服务初始化 | 构造函数注入 |

#### 核心特性

- ✅ **类型安全** - 全面Python类型注解
- ✅ **异步支持** - FastAPI异步端点
- ✅ **结果封装** - Result[T]统一返回
- ✅ **操作审计** - 完整日志记录
- ✅ **降级策略** - LLM不可用时自动降级
- ✅ **线程安全** - 文件存储锁机制
