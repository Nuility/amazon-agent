# Amazon Ads Agent 测试指南

## 测试环境要求

- Python 3.8 或更高版本
- 所有依赖包已安装

## 快速测试

### 方式一：运行自动化测试脚本

双击运行：
```
run_tests.bat
```

或在命令行执行：
```bash
python tests/comprehensive_test.py
```

### 方式二：手动测试

如果自动测试无法运行，请按以下步骤手动测试：

---

## 详细测试步骤

### 测试1：环境检查

```bash
# 检查Python版本
python --version
# 期望输出: Python 3.8.x 或更高

# 检查项目目录
cd agenttest
dir
# 应该看到 src, config, tests 等目录
```

### 测试2：依赖包安装

```bash
# 安装核心依赖
pip install -r requirements.txt

# 验证安装
python -c "import fastapi; print('FastAPI:', fastapi.__version__)"
python -c "import pydantic; print('Pydantic:', pydantic.__version__)"
python -c "import yaml; print('PyYAML: 已安装')"
```

### 测试3：配置文件测试

```bash
# 查看配置文件
type config\config.yaml

# 验证配置格式
python -c "import yaml; print(yaml.safe_load(open('config/config.yaml', encoding='utf-8')))"
```

### 测试4：应用初始化测试

```bash
# 启动CLI查看帮助
python -m agenttest --help
# 或
python src/main.py --help
```

### 测试5：用户管理功能测试

```bash
# 创建用户
python -m agenttest create --username "测试用户" --email "test@example.com"

# 查询用户列表
python -m agenttest list

# 获取统计数据
python -m agenttest stats
```

### 测试6：Web界面测试

```bash
# 启动Web服务器
python start.py

# 或使用uvicorn
cd src/api
python run_server.py
```

访问以下地址验证：
- 主页: http://127.0.0.1:8001
- API文档: http://127.0.0.1:8001/docs
- 健康检查: http://127.0.0.1:8001/api/health

### 测试7：API端点测试

使用浏览器或curl测试：

```bash
# 健康检查
curl http://127.0.0.1:8001/api/health

# 用户列表
curl http://127.0.0.1:8001/api/users

# 聊天交互
curl -X POST http://127.0.0.1:8001/api/chat \
  -H "Content-Type: application/json" \
  -d "{\"message\": \"查询所有用户\"}"

# 统计数据
curl http://127.0.0.1:8001/api/statistics
```

### 测试8：国际化测试

1. 启动Web界面
2. 点击右上角语言选择器
3. 切换到不同语言（English、中文等）
4. 验证界面文本是否正确切换

### 测试9：LLM配置测试

1. 启动Web界面
2. 导航到"大模型配置"
3. 尝试配置不同的LLM提供商
4. 点击"测试连接"验证

## 故障排查

### 问题1：Python命令未找到

**解决方案**：
- 确保Python已安装
- 将Python添加到系统环境变量PATH
- 或使用完整路径：`%LOCALAPPDATA%\Programs\Python\Python314\python.exe`

### 问题2：依赖包安装失败

**解决方案**：
```bash
# 使用国内镜像源
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 问题3：端口8001被占用

**解决方案**：
```bash
# 检查端口占用
netstat -ano | findstr :8001

# 使用其他端口启动
uvicorn src.api.server:app --port 8002
```

### 问题4：配置文件加载失败

**解决方案**：
- 检查config/config.yaml文件是否存在
- 检查YAML格式是否正确
- 检查文件编码是否为UTF-8

### 问题5：数据存储失败

**解决方案**：
- 检查data目录权限
- 确保磁盘空间充足
- 检查users.json文件格式
