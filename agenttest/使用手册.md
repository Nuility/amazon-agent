## 1. 系统简介

Amazon Ads Agent是一个智能化广告优化系统，提供广告数据分析、优化建议生成、关键词排名追踪等功能，支持与大模型API集成实现智能决策。

### 1.1 核心功能

| 功能模块 | 说明 | 版本 |
|---------|------|------|
| **广告数据分析** | 广告活动数据汇总、性能指标计算 | v1.0 |
| **优化建议生成** | 基于LLM的广告优化建议 | v1.0 |
| **关键词排名追踪** | 关键词排名变化监控 | v1.0 |
| **代理工作流执行** | 自动化优化工作流 | v1.0 |
| **用户管理** | 用户CRUD、批量操作 | v1.0 |
| **配置管理** | LLM配置、提示词模板管理 | v1.0 |
| **国际化支持** | 8种语言实时切换 | v2.1 |
| **Skill系统** | 技能注册、查询、执行（类似Codex） | v2.2 ✨ |
| **智能对话增强** | 多轮对话、意图识别、上下文管理 | v2.2 ✨ |
| **WebSocket流式对话** | 实时流式响应 | v2.2 ✨ |
| **更新日志管理** | 标准化的更新日志管理 | v2.2 ✨ |

### 1.2 技术栈

| 类别 | 技术 | 版本 |
|------|------|------|
| **后端框架** | FastAPI | 0.100.0+ |
| **ASGI服务器** | Uvicorn | 0.23.0+ |
| **数据验证** | Pydantic | 2.0.0+ |
| **异步HTTP** | httpx | 0.24.0+ |
| **WebSocket** | websockets | 11.0+ |
| **数据存储** | JSON文件 | - |
| **配置管理** | PyYAML | 6.0+ |
| **前端** | 原生JavaScript + CSS3 | - |
| **国际化** | i18n系统 | 自研 |

### 1.3 系统特性

- ✅ **类型安全** - 全面Python类型注解，Pydantic v2强类型验证
- ✅ **分层架构** - 表现层/应用层/领域层/基础设施层
- ✅ **降级策略** - LLM不可用时自动降级到规则引擎
- ✅ **操作审计** - 完整日志记录
- ✅ **线程安全** - 文件存储锁机制
- ✅ **国际化** - 8种语言支持
- ✅ **响应式UI** - 适配桌面和移动端
- ✅ **Skill系统** - 类似Codex的技能管理功能（v2.2新增）
- ✅ **智能对话** - 类似ChatGPT的多轮对话能力（v2.2新增）
- ✅ **流式响应** - WebSocket实时流式对话（v2.2新增）
- ✅ **意图识别** - LLM + 规则引擎双重识别（v2.2新增）

## 4. 配置说明

### 4.1 配置文件位置

| 文件 | 路径 | 说明 |
|------|------|------|
| 主配置 | `config/config.yaml` | 系统主配置 |
| 提示词配置 | `config/ad_agent_prompt.json` | 广告代理提示词 |
| 环境变量 | `.env` | 环境变量配置（从.env.example复制） |

### 4.2 主配置项说明

```yaml
# config/config.yaml

# 批量操作配置
max_batch_size: 1000

# 大模型API配置
enable_llm_integration: true
llm_api_config:
  provider: "minimax"          # openai/pangu/minimax/mock
  api_key: "your-api-key"
  api_endpoint: "https://api.minimaxi.com/anthropic"
  model: "MiniMax-M2.7"
  timeout: 30
  max_retries: 3

# Amazon Ads API 配置
amazon_ads_config:
  enabled: false
  region: "na"                 # na/eu/fe
  client_id: ""
  client_secret: ""
  refresh_token: ""
  profile_id: ""
  api_program: "report"
  access_level: "view"

# 日志配置
log_level: "INFO"              # DEBUG/INFO/WARNING/ERROR
log_file_path: "./logs/operation.log"
log_max_size: 10485760         # 10MB
log_backup_count: 5

# 存储配置
data_storage_type: "file"      # file/sqlite
data_file_path: "./data/users.json"
sqlite_db_path: "./data/users.db"

# 性能配置
cache_enabled: true
cache_ttl: 300
connection_pool_size: 10

# 安全配置
sensitive_fields:
  - "password"
  - "token"
  - "secret"
audit_enabled: true
```

## 8. Skill系统使用指南（v2.2新增）

### 8.1 什么是Skill系统？

Skill系统类似Codex，提供可注册、可查询、可执行的技能管理功能。每个Skill都有明确的参数定义和执行逻辑。

### 8.2 预置Skill列表

| Skill ID | 名称 | 说明 |
|----------|------|------|
| `ad_analysis` | 广告分析 | 分析ACOS、ROAS、CTR等关键指标 |
| `keyword_query` | 关键词查询 | 查询关键词表现数据 |
| `optimization_suggest` | 优化建议生成 | 生成竞价调整等优化建议 |

### 8.3 使用Skill

**通过API**:

```bash
# 1. 查看可用Skill
curl http://localhost:8001/api/v1/skills

# 2. 查看Skill详情
curl http://localhost:8001/api/v1/skills/ad_analysis

# 3. 查看Skill帮助
curl http://localhost:8001/api/v1/skills/ad_analysis/help

# 4. 执行Skill
curl -X POST http://localhost:8001/api/v1/skills/ad_analysis/execute \
  -H "Content-Type: application/json" \
  -d '{
    "params": {
      "campaign_ids": ["camp001", "camp002"],
      "date_range": {"start": "2026-01-01", "end": "2026-05-19"}
    }
  }'
```

**通过JavaScript**:

```javascript
// 执行Skill
async function executeSkill(skillId, params) {
    const response = await fetch(`/api/v1/skills/${skillId}/execute`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ params })
    });
    return await response.json();
}

// 使用示例
const result = await executeSkill('ad_analysis', {
    campaign_ids: ['camp001', 'camp002']
});
```

```javascript
    ws.send(JSON.stringify({ content: '帮我优化广告竞价' }));
};

ws.onmessage = (event) => {
    const data = JSON.parse(event.data);
    console.log('收到消息:', data);
};
```

### 9.3 支持的意图类型

| 意图 | 说明 | 示例 |

## 13. 常见问题FAQ

### Q1: 如何修改数据存储位置？

**A**: 在配置文件中修改 `data_file_path`，或设置环境变量 `AGENTTEST_DATA_FILE_PATH`。

### Q2: 如何启用大模型智能分析？

**A**: 
1. 在配置文件中设置 `enable_llm_integration: true`
2. 配置 `llm_api_config` 中的 `api_key` 和 `api_endpoint`
3. 选择提供商（openai/pangu/minimax）

### Q3: 批量导入失败怎么办？

**A**: 
1. 检查文件格式是否正确（JSON或CSV）
2. 检查数据是否符合字段要求（用户名、邮箱必填）
3. 查看日志文件 `logs/operation.log` 获取详细错误信息

### Q4: 如何查看操作日志？

**A**: 日志文件位于 `logs/operation.log`，包含所有操作的审计记录。

### Q5: 系统支持哪些用户状态？

**A**: 支持三种状态：
- `active`：活跃状态
- `inactive`：非活跃状态
- `deleted`：已删除状态

### Q6: 如何备份数据？

**A**: 直接复制数据文件 `data/users.json` 即可。

### Q7: Web界面无法访问怎么办？

**A**: 
1. 检查服务是否启动：查看命令行是否有输出
2. 检查端口是否被占用：`netstat -ano | findstr :8001`
3. 尝试其他端口：修改启动脚本中的端口配置
4. 检查防火墙设置：确保端口允许访问

### Q8: LLM调用失败怎么办？

**A**:
1. 检查API密钥是否正确
2. 检查网络连接是否正常
3. 查看日志获取详细错误信息
4. 系统会自动降级到本地规则引擎

### Q9: 如何在生产环境部署？

**A**: 
推荐使用Gunicorn + Nginx反向代理：

```bash
pip install gunicorn

gunicorn server:app \
    --workers 4 \
    --worker-class uvicorn.workers.UvicornWorker \
    --bind 0.0.0.0:8001
```

### Q10: 广告代理运行时间过长？

**A**:
- LLM API响应时间通常在5-30秒
- 如果数据量大，可能需要更长时间
- 可以减少 `max_recommendations` 参数
- 检查LLM API的超时配置
