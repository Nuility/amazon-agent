## 1. 系统简介

Amazon Ads Agent 是一个智能化广告优化系统，集成了大模型AI能力，提供Amazon广告活动的智能分析、优化建议生成和自动化执行能力。系统提供两种交互方式：**命令行接口（CLI）** 和 **Web图形界面（UI）**。

### 1.1 核心功能

- **用户管理**：创建、查询、更新、删除用户
- **批量操作**：批量创建、批量更新、批量删除、文件导入
- **广告数据洞察**：广告活动分析、关键词排名、搜索词分析
- **AI智能优化**：大模型驱动的广告优化建议生成
- **Amazon API集成**：直接对接Amazon Ads API获取实时数据
- **配置管理**：配置加载、验证、热更新
- **国际化支持**：支持8种语言界面（中文、英文、西班牙语、法语、德语、葡萄牙语、日语、阿拉伯语）

### 1.2 系统特性

- ✅ 类型安全（Python类型注解）
- ✅ 可扩展架构（插件系统）
- ✅ 降级策略（API失败自动降级）
- ✅ 操作审计（完整日志记录）
- ✅ 数据脱敏（敏感信息保护）
- ✅ 双交互模式（CLI + Web UI）
- ✅ 多LLM支持（OpenAI、华为云盘古、MiniMax）
- ✅ 国际化（8种语言）

## 4. 配置说明

### 4.1 配置文件格式

系统支持 YAML 和 JSON 格式的配置文件，默认使用 `config/config.yaml`。

### 4.2 配置文件位置

- 默认配置文件：`config/config.yaml`
- Prompt模板配置：`config/ad_agent_prompt.json`
- 环境变量配置：`.env`（需从 `.env.example` 复制）

### 4.3 核心配置项说明

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `max_batch_size` | int | 1000 | 批量操作最大数量限制 |
| `enable_llm_integration` | bool | true | 是否启用大模型集成 |
| `llm_api_config` | dict | {} | 大模型API配置 |
| `amazon_ads_config` | dict | {} | Amazon Ads API配置 |
| `log_level` | string | INFO | 日志级别 |
| `log_file_path` | string | ./logs/operation.log | 日志文件路径 |
| `data_storage_type` | string | file | 存储类型 |
| `data_file_path` | string | ./data/users.json | 数据文件路径 |
| `cache_enabled` | bool | true | 是否启用缓存 |
| `audit_enabled` | bool | true | 是否启用审计日志 |

### 4.4 大模型API配置

#### 4.4.1 OpenAI配置示例

```yaml
enable_llm_integration: true
llm_api_config:
  provider: "openai"
  api_key: "sk-your-api-key-here"
  api_endpoint: "https://api.openai.com/v1"
  model: "gpt-3.5-turbo"
  timeout: 30
  max_retries: 3
```

#### 4.4.2 华为云盘古配置示例

```yaml
enable_llm_integration: true
llm_api_config:
  provider: "pangu"
  api_key: "your-api-key-here"
  api_endpoint: "https://your-pangu-endpoint"
  model: "pangu-model"
  timeout: 30
  max_retries: 3
```

#### 4.4.3 MiniMax配置示例（推荐）

```yaml
enable_llm_integration: true
llm_api_config:
  provider: "minimax"
  api_key: "sk-..."
  api_endpoint: "https://api.minimaxi.com/anthropic"
  model: "MiniMax-M2.7"
  timeout: 30
  max_retries: 3
```

### 4.5 Amazon Ads API配置

```yaml
amazon_ads_config:
  enabled: true
  region: "na"                    # na=北美, eu=欧洲, fe=远东
  client_id: "amzn1.application-oa2-client..."
  client_secret: "..."

  profile_id: "..."
```

### 4.6 环境变量配置

复制 `.env.example` 为 `.env` 并修改：

```bash
cp .env.example .env
```

环境变量说明：

| 环境变量 | 说明 |
|----------|------|
| `AGENTTEST_MAX_BATCH_SIZE` | 批量操作限制 |
| `AGENTTEST_LOG_LEVEL` | 日志级别 |
| `AGENTTEST_LLM_API_KEY` | 大模型API密钥 |
| `AGENTTEST_LLM_API_ENDPOINT` | 大模型API端点 |
| `AGENTTEST_LLM_MODEL` | 大模型名称 |
| `AGENTTEST_STORAGE_TYPE` | 存储类型 |
| `AGENTTEST_DATA_FILE_PATH` | 数据文件路径 |

**注意**：环境变量优先级高于配置文件。

```javascript
// 聊天交互
async function chat(message) {
    const response = await fetch('http://localhost:8001/api/chat', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ message })
    });
    return await response.json();
}

// 获取用户列表
async function getUsers(page = 1, pageSize = 20) {
    const response = await fetch(
        `http://localhost:8001/api/users?page=${page}&page_size=${pageSize}`

// 运行AI智能体
async function runAgent(objective, dryRun = true) {
    const response = await fetch('http://localhost:8001/api/ad-agent/run', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ objective, dry_run: dryRun })
    });
    return await response.json();
}
```

## 10. 常见问题FAQ

### Q1: 如何修改数据存储位置？

**A**: 在配置文件中修改 `data_file_path`，或设置环境变量 `AGENTTEST_DATA_FILE_PATH`。

### Q2: 如何启用大模型智能分析？

**A**: 
1. 在配置文件中设置 `enable_llm_integration: true`
2. 配置 `llm_api_config` 中的 `provider`、`api_key` 和 `api_endpoint`
3. 或在Web界面的"LLM配置"页面可视化配置

### Q3: 支持哪些大模型提供商？

**A**: 支持4种提供商：
- OpenAI (GPT-3.5/GPT-4)
- 华为云盘古
- MiniMax (M2系列)
- Mock客户端（测试用）

### Q4: 批量导入失败怎么办？

**A**: 
1. 检查文件格式是否正确（JSON或CSV）
2. 检查数据是否符合字段要求（用户名、邮箱必填）
3. 查看日志文件 `logs/operation.log` 获取详细错误信息

### Q5: 如何查看操作日志？

**A**: 日志文件位于 `logs/operation.log`，包含所有操作的审计记录。

### Q6: 系统支持哪些用户状态？

**A**: 支持三种状态：
- `active`：活跃状态
- `inactive`：非活跃状态
- `deleted`：已删除状态

### Q7: 如何备份数据？

**A**: 直接复制数据文件 `data/users.json` 即可。

### Q8: Web界面无法访问怎么办？

**A**: 
1. 检查服务是否启动：查看命令行是否有输出
2. 检查端口是否被占用：`netstat -ano | findstr :8001`（Windows）
3. 尝试其他端口：修改启动脚本中的端口配置
4. 检查防火墙设置：确保端口允许访问

### Q9: pip命令未找到怎么办？

**A**: Python已安装但未添加到环境变量，解决方案：

**方式一：使用完整路径**
```bash
%LOCALAPPDATA%\Programs\Python\Python314\python.exe -m pip install -r requirements.txt
```

**方式二：添加环境变量**
1. 打开"系统属性" → "环境变量"
2. 在Path中添加Python安装路径
3. 重启命令行窗口

**方式三：使用启动脚本**
直接双击 `src/api/start.bat`，脚本会自动检测Python路径

### Q10: 如何切换界面语言？

**A**: 
1. 点击右上角语言选择器
2. 选择目标语言
3. 界面立即切换，无需刷新
4. 语言偏好自动保存

### Q11: 如何配置Amazon Ads API？

**A**: 
1. 在Amazon开发者平台创建应用，获取client_id和client_secret
2. 完成OAuth授权流程，获取refresh_token
3. 在配置文件中配置 `amazon_ads_config`
4. 设置正确的区域（na/eu/fe）

### Q12: AI智能体的dry_run模式是什么？

**A**: 
dry_run模式（试运行模式）下，智能体会：
1. 生成优化建议
2. 验证建议的可行性
3. 但不实际执行修改操作
4. 适合先预览结果再决定是否执行

## 12. 最佳实践

### 12.1 性能优化

- 批量操作优于单个操作
- 使用合适的批量大小（建议500-1000）
- 定期清理已删除用户
- Web界面分页加载，避免一次性加载大量数据
- 启用缓存减少重复计算

### 12.2 安全建议

- 定期备份数据文件
- 保护配置文件中的敏感信息（API密钥等）
- 使用环境变量管理API密钥
- 生产环境配置HTTPS
- 配置CORS白名单

### 12.3 运维建议

- 定期查看日志文件
- 监控异常检测结果
- 及时处理优化建议
- 生产环境使用Gunicorn多进程部署
- 配置Nginx反向代理和SSL证书
- 设置日志轮转防止磁盘占满
