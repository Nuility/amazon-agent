# 用户管理智能体系统 - 使用手册

## 1. 系统简介

用户管理智能体系统是一个智能化、可扩展的用户管理解决方案，提供用户管理、批量操作、智能分析等功能，支持与大模型API集成实现智能化决策。系统提供两种交互方式：**命令行接口（CLI）** 和 **Web图形界面（UI）**。

### 1.1 核心功能

- **用户管理**：创建、查询、更新、删除用户
- **批量操作**：批量创建、批量更新、批量删除、文件导入
- **智能分析**：统计分析、异常检测、优化建议
- **配置管理**：配置加载、验证、热更新
- **Web界面**：现代化图形操作界面，支持聊天式交互
- **大模型集成**：预留OpenAI、华为云盘古等大模型API接口

### 1.2 系统特性

- 类型安全（Python类型注解）
- 可扩展架构（插件系统）
- 降级策略（API失败自动降级）
- 操作审计（完整日志记录）
- 数据脱敏（敏感信息保护）
- 双交互模式（CLI + Web UI）

## 4. 配置说明

### 4.1 配置文件格式

系统支持 YAML 和 JSON 格式的配置文件，默认使用 `config/config.yaml`。

### 4.2 配置文件位置

- 默认配置文件：`config/config.yaml`
- 环境变量配置：`.env`（需从 `.env.example` 复制）

### 4.3 配置项说明

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `max_batch_size` | int | 1000 | 批量操作最大数量限制 |
| `enable_llm_integration` | bool | false | 是否启用大模型集成 |
| `llm_api_config` | dict | {} | 大模型API配置 |
| `enable_hot_reload` | bool | true | 是否启用配置热更新 |
| `log_level` | string | INFO | 日志级别（DEBUG/INFO/WARNING/ERROR） |
| `log_file_path` | string | ./logs/operation.log | 日志文件路径 |
| `data_storage_type` | string | file | 存储类型（file/sqlite） |
| `data_file_path` | string | ./data/users.json | 数据文件路径 |

### 4.4 大模型API配置

#### 4.4.1 OpenAI配置示例

```yaml
enable_llm_integration: true
llm_api_config:
  provider: "openai"
  api_key: "your-api-key-here"
  api_endpoint: "https://api.openai.com/v1"
  model: "gpt-3.5-turbo"
  timeout: 30
  max_retries: 3
```

#### 4.4.2 华为云盘古配置示例

```yaml
enable_llm_integration: true
llm_api_config:
  provider: "pangu"
  api_key: "your-api-key-here"
  api_endpoint: "https://your-pangu-endpoint"
  model: "pangu-model"
  timeout: 30
  max_retries: 3
```

### 4.5 环境变量配置

复制 `.env.example` 为 `.env` 并修改：

```bash
cp .env.example .env
```

环境变量说明：

| 环境变量 | 说明 |
|----------|------|
| `AGENTTEST_MAX_BATCH_SIZE` | 批量操作限制 |
| `AGENTTEST_LOG_LEVEL` | 日志级别 |
| `AGENTTEST_LLM_API_KEY` | 大模型API密钥 |
| `AGENTTEST_LLM_API_ENDPOINT` | 大模型API端点 |
| `AGENTTEST_LLM_MODEL` | 大模型名称 |
| `AGENTTEST_STORAGE_TYPE` | 存储类型 |
| `AGENTTEST_DATA_FILE_PATH` | 数据文件路径 |

**注意**：环境变量优先级高于配置文件。

## 6. Web界面详解

### 6.1 API接口说明

Web界面基于FastAPI构建，提供完整的RESTful API：

| 接口 | 方法 | 说明 | 示例 |
|------|------|------|------|
| `/` | GET | 返回UI页面 | `GET http://localhost:8001/` |
| `/api/health` | GET | 健康检查 | `GET http://localhost:8001/api/health` |
| `/api/chat` | POST | 聊天式交互 | `POST {"message": "查询所有用户"}` |
| `/api/execute` | POST | 执行具体命令 | `POST {"command": "create_user", "params": {...}}` |
| `/api/users` | GET | 获取用户列表 | `GET /api/users?page=1&page_size=20` |
| `/api/users` | POST | 创建用户 | `POST {"username": "...", "email": "..."}` |
| `/api/users/{id}` | GET | 获取用户详情 | `GET /api/users/xxx` |
| `/api/users/{id}` | PUT | 更新用户 | `PUT {"username": "新名字"}` |
| `/api/users/{id}` | DELETE | 删除用户 | `DELETE /api/users/xxx` |
| `/api/statistics` | GET | 获取统计数据 | `GET /api/statistics` |
| `/api/config` | GET | 获取配置信息 | `GET /api/config` |

### 6.2 API调用示例

#### 使用curl调用

```bash
# 健康检查
curl http://localhost:8001/api/health

# 聊天交互
curl -X POST http://localhost:8001/api/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "查询所有用户"}'

# 创建用户
curl -X POST http://localhost:8001/api/users \
  -H "Content-Type: application/json" \
  -d '{"username": "测试用户", "email": "test@example.com"}'

# 查询用户列表
curl "http://localhost:8001/api/users?page=1&page_size=10"

# 更新用户
curl -X PUT http://localhost:8001/api/users/{user_id} \
  -H "Content-Type: application/json" \
  -d '{"username": "新用户名"}'

# 删除用户
curl -X DELETE http://localhost:8001/api/users/{user_id}
```

#### 使用JavaScript调用

```javascript
// 聊天交互
async function chat(message) {
    const response = await fetch('http://localhost:8001/api/chat', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ message })
    });
    return await response.json();
}

// 获取用户列表
async function getUsers(page = 1, pageSize = 20) {
    const response = await fetch(
        `http://localhost:8001/api/users?page=${page}&page_size=${pageSize}`
    );
    return await response.json();
}

// 创建用户
async function createUser(userData) {
    const response = await fetch('http://localhost:8001/api/users', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(userData)
    });
    return await response.json();
}
```


## 7. 常见问题FAQ

### Q1: 如何修改数据存储位置？

**A**: 在配置文件中修改 `data_file_path`，或设置环境变量 `AGENTTEST_DATA_FILE_PATH`。

### Q2: 如何启用大模型智能分析？

**A**: 
1. 在配置文件中设置 `enable_llm_integration: true`
2. 配置 `llm_api_config` 中的 `api_key` 和 `api_endpoint`
3. 参考"大模型API配置"章节完成具体配置

### Q3: 批量导入失败怎么办？

**A**: 
1. 检查文件格式是否正确（JSON或CSV）
2. 检查数据是否符合字段要求（用户名、邮箱必填）
3. 查看日志文件 `logs/operation.log` 获取详细错误信息

### Q4: 如何查看操作日志？

**A**: 日志文件位于 `logs/operation.log`，包含所有操作的审计记录。

### Q5: 系统支持哪些用户状态？

**A**: 支持三种状态：
- `active`：活跃状态
- `inactive`：非活跃状态
- `deleted`：已删除状态

### Q6: 如何备份数据？

**A**: 直接复制数据文件 `data/users.json` 即可。

### Q7: Web界面无法访问怎么办？

**A**: 
1. 检查服务是否启动：查看命令行是否有输出
2. 检查端口是否被占用：`netstat -ano | findstr :8001`
3. 尝试其他端口：修改 `run_server.py` 中的端口配置
4. 检查防火墙设置：确保端口允许访问

### Q8: pip命令未找到怎么办？

**A**: 
Python已安装但未添加到环境变量，解决方案：

**方式一：使用完整路径**
```bash
%LOCALAPPDATA%\Programs\Python\Python314\python.exe -m pip install -r requirements.txt
```

**方式二：添加环境变量**
1. 打开"系统属性" → "环境变量"
2. 在Path中添加Python安装路径
3. 重启命令行窗口

**方式三：使用启动脚本**
直接双击 `src/api/start.bat`，脚本会自动检测Python路径

### Q9: 如何在生产环境部署Web界面？

**A**: 参考"Web界面详解 → 生产部署"章节，推荐使用Gunicorn + Nginx反向代理。

### Q10: API返回404怎么办？

**A**: 
1. 确认API路径正确（需要加 `/api` 前缀）
2. 访问 http://localhost:8001/docs 查看可用接口
3. 检查请求方法是否正确（GET/POST/PUT/DELETE）

## 8. 错误代码说明

| 错误码 | 说明 | 解决方案 |
|--------|------|----------|
| 1001 | 用户不存在 | 检查用户ID是否正确 |
| 1002 | 用户已存在 | 使用其他用户ID或更新现有用户 |
| 1003 | 数据校验失败 | 检查数据格式是否符合要求 |
| 1004 | 存储错误 | 检查文件权限、磁盘空间 |
| 1005 | 配置错误 | 检查配置文件格式和内容 |
| 1006 | 大模型API错误 | 检查API配置和网络连接 |
| 1007 | 批量操作超限 | 减少批量操作数量 |
| 1008 | 无效状态转换 | 检查状态流转规则 |
| 1010 | 操作失败 | 查看日志获取详细信息 |
