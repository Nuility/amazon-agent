## 4. 扩展开发指南

### 4.1 插件开发规范

#### 插件接口

```python
from abc import ABC, abstractmethod
from typing import Dict, Any

class PluginInterface(ABC):
    """插件接口规范"""
    
    @property
    @abstractmethod
    def name(self) -> str:
        """插件名称"""
        pass
    
    @property
    @abstractmethod
    def version(self) -> str:
        """插件版本"""
        pass
    
    @abstractmethod
    def initialize(self, config: Dict[str, Any]) -> bool:
        """初始化插件"""
        pass
    
    @abstractmethod
    def execute(self, context: Dict[str, Any]) -> Any:
        """执行插件逻辑"""
        pass
    
    @abstractmethod
    def cleanup(self) -> None:
        """清理资源"""
        pass
```

#### 插件开发步骤

1. 继承 `PluginInterface` 类
2. 实现所有抽象方法
3. 创建插件配置文件 `plugin.yaml`
4. 编写插件测试
5. 编写插件文档
6. 提交到插件市场

### 7.3 提交信息规范

遵循Conventional Commits规范：

```
<type>(<scope>): <subject>

<body>

<footer>
```

类型（type）:
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 重构
- `perf`: 性能优化
- `test`: 测试相关
- `chore`: 构建/工具相关
- `ci`: CI/CD配置

示例：
```

- Add batch execution method to AdAgentService
- Implement execution queue management
- Add progress tracking

Closes #123
```
